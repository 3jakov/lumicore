What counts as a finding:
- integration flaw
- state/query issue
- contract mismatch risk
- bad temporary abstraction
- docs violation
- structure that will make next project steps messy

What NOT to include:

- visual preference feedback
- backend review
- generic placeholder criticism if current scope is acceptable

Format:
1. Findings first, ordered by severity
2. For each finding:
